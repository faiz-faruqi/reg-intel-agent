Architecture decision records

Five decisions that define the current shape of the system and its production transition path.

ADR-001 Accepted
LangGraph vs. Amazon Bedrock Agents
Decision Use LangGraph with a supervisor + specialist pattern for deterministic step ordering and explicit interrupt points. Context Bedrock Agents is managed with opaque routing. A compliance audit trail must be inspectable and reproducible — LangGraph exposes the full graph. Trade-off Bedrock Agents reduces ops burden and is preferred when the team lacks ML platform capacity. LangGraph requires maintaining graph code.
ADR-002 Accepted
pgvector on Neon (demo) vs. dedicated vector DB (production)
Decision Co-locate the vector store in the same Neon PostgreSQL instance as the audit log. Single connection string, zero incremental cost at demo scale. Context 15 chunks, ~2 QPS. A dedicated Pinecone cluster adds $70+/month with no latency benefit. Neon handles this comfortably at zero incremental cost. Trade-off Migration trigger: corpus exceeds ~1M chunks, QPS > 50, or SLA requires sub-10ms P99. Then: Aurora Serverless v2 + pgvector or OpenSearch Serverless.
ADR-003 Accepted
Application guardrails vs. Bedrock Guardrails
Decision Implement application-level guardrails (src/guardrails.py) mirroring the Bedrock Guardrails API. Production swap is a config-only change. Context Bedrock Guardrails requires AWS. Demo runs on Railway. Mirrored API surface means business logic is identical in both environments. Trade-off Bedrock Guardrails adds managed PII redaction, groundedness checks, and policy enforcement at the inference layer — preferred for production OSFI E-23 alignment.
ADR-004 Accepted
Railway (demo) vs. AWS App Runner / ECS Fargate (production)
Decision Deploy to Railway. Auto-deploys from main branch, ~$5–10/month compute. Services sleep after 30 min inactivity — acceptable for demo. Context No real client traffic, no SLA. Railway removes all infra overhead — right for an interview artifact at effectively zero marginal cost. Trade-off Production trigger: a real engagement, private networking requirement, or SLA. App Runner (simpler) or ECS Fargate (full control) — IAM role replaces API key.
ADR-005 Production path
OpenRouter (demo) vs. Amazon Bedrock (production)
Decision Use OpenRouter with google/gemma-4-31b-it:free for generation and text-embedding-3-small for embeddings. Production swaps to Bedrock — no application code changes. Context Generation cost = $0 at demo volume. Same ChatOpenAI client class works with a different base_url — OpenAI SDK compatibility is the critical design choice. Trade-off Bedrock requires AWS account, 1024-dim embeddings (rebuild HNSW index), and IAM role assignment. Data stays in-region and OSFI E-23 MDS compliance is cleaner.
