Architecture

What Happens When You Submit a Query

A LangGraph supervisor routes your question through three specialist agents. Every step is traced in LangSmith and appended to an immutable PostgreSQL audit log.

🛡️
Input Guardrail
Screens for denied topics (hacking, evasion, harmful content) and PII patterns before any agent sees the question.
🔍
Knowledge Agent
Embeds the question and runs cosine similarity search via pgvector HNSW index over seeded regulatory docs (GDPR, Basel III, OSFI).
✍️
Analysis Agent
Drafts a compliance-grounded response using retrieved chunks. Citations are mandatory — un-cited claims are flagged, not emitted.
👤
Human Approval Gate
Any write action (Jira ticket, GitHub issue) is proposed and the graph pauses. A human must explicitly approve before any write executes.
📋
Audit Log
Every step, tool call, and approval appended to PostgreSQL. DELETE and UPDATE are blocked by a database-level trigger — tamper-proof by design.
Under the Hood

Step-by-Step Execution

What the code actually does when a question arrives at the API.

1
Supervisor routes the question
A LangGraph StateGraph with a supervisor node checks state on each transition and routes: knowledge → analysis → (action in CLI mode). Uses AgentState(TypedDict, total=False) so partial returns merge safely without overwriting existing state keys.
2
pgvector semantic retrieval
The question is embedded with text-embedding-3-small (1536-dim). An HNSW index on the documents table returns top-k chunks by cosine similarity. HNSW works at any dataset size — unlike IVFFlat, which requires hundreds of rows to be useful.
3
Citation-mandatory drafting
Analysis Agent receives retrieved chunks and is prompted: every factual claim must reference a [N] citation. The response is checked for is_cited=True before emission — hallucination is structurally constrained, not just prompted against.
4
Output guardrail
The draft is screened for PII patterns (SSN regex, credit card regex, email) and harmful content before being returned to the caller. API-compatible with AWS Bedrock Guardrails — swapping to managed guardrails in production is a config change, not a rewrite.
5
Action proposal + HITL gate (CLI mode)
Action Agent generates a JSON proposal {title, body, labels} for a Jira ticket or GitHub issue. LangGraph's interrupt_before=["human_gate"] pauses the graph. The CLI prompts for approve/reject, then calls graph.update_state() and resumes — no write without human intent.
6
Append-only audit trail
Every agent step and human decision is written to the audit_log table in Neon PostgreSQL with a UTC timestamp. A BEFORE DELETE and BEFORE UPDATE trigger raises an exception on any modification attempt — immutability enforced at the DB level.
Governance

Built for Regulated Environments

Six non-negotiable pillars that make this system auditable, safe, and honest. Each maps directly to a production AWS equivalent — described in the ADRs.

🛡️
Bedrock Guardrails (mirrored)
Application-level guardrails screen every input and output for denied topics, PII, and harmful content. API-compatible with AWS Bedrock Guardrails — production swap is a single config change.
📋
Immutable Audit Log
Every agent step, tool call, and human approval is appended to PostgreSQL. A database-level trigger prevents any DELETE or UPDATE. Regulators can trust the log is tamper-proof — not just by convention.
👤
Human-in-the-Loop Gate
The Action Agent proposes write actions — it never executes them. LangGraph's interrupt_before pauses the graph until a human explicitly approves. Rejection is also logged.
📎
Citations Mandatory
Every factual claim must reference a numbered source. Un-cited responses are flagged before emission. Hallucination is constrained at the prompt and enforcement layer — both.
🔑
IAM Least-Privilege
The Bedrock IAM user is scoped to bedrock:InvokeModel on target model ARNs only. In production (AWS compute), this becomes an IAM role — no long-lived access keys at all.
🔭
Full Observability
Every agent step and LLM call is traced in LangSmith. The full graph — supervisor routing, retrieval results, draft generations — is visible, replayable, and attributable to a specific run.

Infrastructure

Two-Tier Deployment Strategy

Deliberately cost-optimised for demo portability at ~CAD $20–35/month. The production path is fully designed in ADRs — moving to AWS changes infrastructure, not application code.

● Demo Tier — Active Now
  • Railway Hobby (FastAPI API)
  • Neon free tier (pgvector)
  • OpenRouter → Claude Sonnet
  • LangSmith tracing
  • ~CAD $20–35/month total
  • Sleeps after 30 min idle
● Production Path — ADR-004
  • AWS App Runner / ECS Fargate
  • Aurora Serverless v2 + pgvector
  • Amazon Bedrock (IAM role)
  • AWS Secrets Manager
  • Private VPC + WAF
  • Always-on, private networking
● Compliance Alignment
  • OSFI E-23 model risk
  • GDPR Article 5 data minimisation
  • Basel III capital adequacy
  • Append-only audit trail
  • Bedrock Guardrails
